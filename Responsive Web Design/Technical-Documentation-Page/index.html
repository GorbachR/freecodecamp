Installation

Npm package manager

The most common way to install bootstrap is to use a package manager like npm to download the source code and its dependencies. Simply run the following command after you have initialized npm:

npm install bootstrap

To then start using bootstrap you need to load bootstrap into your HTML file. Copy the following link tag into the head of your HTML file:

<link rel="stylesheet" href="./node_modules/bootstrap/dist/css/bootstrap.min.css" />

Bootstrap requires dependencies like Popper, jquery and its own JavaScript plugin for some features to function properly. To load them include this at the end of your body:

<script src="/node_modules/bootstrap/dist/js/bootstrap.bundle.js"></script>

CDN installation

The easiest way to get started with bootstrap is to use the CDN installation. Simply copy the following into the head of your HTML:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-EVSTQN3/azprG1Anm3QDgpJLIm9Nao0Yz1ztcQTwFspd3yD65VohhpuuCOmLASjC" crossorigin="anonymous">

Also copy this at the bottom of your body for JavaScript functionality:

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/js/bootstrap.bundle.min.js" integrity="sha384-MrcW6ZMFYlzcLA8Nl+NtUVF0sA7MsXsP1UyJoMp4YLEuNSfAP+JcXn/tWtIaxVXM" crossorigin="anonymous"></script>

If you want to install Popper seperately from Boostrap's compiled JavaScript include the following <script> tags:

<script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.9.2/dist/umd/popper.min.js" integrity="sha384-IQsoLXl5PILFhosVNubq5LC7Qb9DXgDA9i+tQ8Zj3iwWAwPtgFTxbJ8NT4GN1R8p" crossorigin="anonymous"></script> <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/js/bootstrap.min.js" integrity="sha384-cVKIPhGWiC2Al4u+LWgxfKTRIcfu0JTxR+EQDz/bgldoEyl4H0zUF0QKbrJ0EcQF" crossorigin="anonymous"></script>
Layout

Containers

Containers are the most basic layout element required to use the bootstrap grid system. To start using them you have three options to choose from:

  • .container with which you can set a max-width at each responsive breakpoint
  • .container-fluid which uses width: 100% at every breakpoint
  • .container-(breakpoint) with which you set the width: 100% until a specific breakpoint

Responsive Breakpoints

Bootstrap uses multiple media queries to properly scale up the elements and ensure a responsive layout. Since bootstrap is mobile first the standard viewport width is up to 575px.


// Extra small devices (portrait phones, less than 576px)
// No media query for `xs` since this is the default in Bootstrap

// Small devices (landscape phones, 576px and up, shorthand: sm)
@media (min-width: 576px) { ... }

// Medium devices (tablets, 768px and up, shorthand: md)
@media (min-width: 768px) { ... }

// Large devices (desktops, 992px and up, shorthand: lg)
@media (min-width: 992px) { ... }

// Extra large devices (large desktops, 1200px and up, shorthand: xl)
@media (min-width: 1200px) { ... }

Grid

The bootstrap grid system uses containers, rows and columns to align content. It is built with flexbox and fully responsive. Here is an example of a simple grid layout: 


<div class="container">
    <div class="row">
        <div class="col-sm">
        One of three columns
        </div>
        <div class="col-sm">
        One of three columns
        </div>
        <div class="col-sm">
        One of three columns
        </div>
    </div>
</div>

Containers help to center and horizontally pad the content. Rows wrap columns, which are the only immediate children of rows and contain the content. With column classes you can decide how many colums an element should take up out of a maximum of 12. Since column widths are set with percentages they are responsive and change relative to the container width.

Setting column width

Bootstrap will automatically set the width of grid columns if they are not specified. You can however change how many of the 12 columns a column element should take up with the col-{number of columns up to 12} class assigned to the column. If you only assigned it to one column element Boostrap will automatically fill up the rest of the row with the other elements.

Setting column widths based on breakpoints

A more common way to to utilize the technique described above is to combine it with bootstrap breakpoints. With the class col-{breakpoint}-{number of column} you can set the width of each individual column item based on the viewport (Breakpoints as described above). With col-{breakpoint}-auto the column will be sized based on its content.

Alignment of Grid

Alignment classes allow you to horizontally and vertically position your rows/ columns anywhere in the container:

Vertical alignment
  • With align-items-start/center/end you can vertically align rows however you want.
  • With align-self-start/center/end you can vertically align columns however you want
Horizontal alignment

With justify-content-start/center/end you can horizontally align rows however you want.

Gutters

Gutters are the space between columns to space the content in the grid apart. At the start and end of row this padding is offset with negative margins to ensure that the columns are properly lined up. You can choose a value between 0 and 5 for your gutters.

  • With .gx-* classes are used to set the horizontal gutter widths.
  • With .gy-* classes are used to set the vertical gutter widths.

To get rid of gutters altogether you can use the .no-gutters class on the row element.

Content

Typography

Bootstrap by default uses its own standard typography, display and link styles. By adjusting the scss variables you can change these custom styles (For more information click here). Responsive typography is part of the standard styling, which adjusts the font size based on viewport size.

Headings and display headings

Headings from <h1>-<h6> are available as usual. Boostrap offers the option to make any element look like a heading. Just use the .h1-.h6 class. If you need a heading that looks different and stands out you could use display headings instead. Add them with the .display-1 through .display-4 classes.

Inline text elements

Bootstrap adds its own style to the following inline HTML elements

  • mark
  • del
  • s
  • ins
  • u
  • small (Also available as class: .small)
  • strong (Also available as class: .strong)
  • em

Other text stylings

  • abbr (Add full text with the title attribute)
  • Blockquotes
    (By using <blockquote> element with the .blockquote class). You can add a <footer> and include a <cite> element to it to add a source.

Lists

You have the option to remove the standard list-style and left margin with the .list-unstyled class. This only counts for the immediate children of the list.

Components

Buttons

Bootstrap also offers predefined button styles that you can use with the utility class .btn in combination with other utility like .btn-primary, .btn-secondary, .btn-success and so on. These styles are also available as an outline version with the utility classes .btn-outline-primary, .btn-outline-secondary etc... For more information click here.

Sizes

There are also two pre-defined button sizes that you can set with the .btn-lg or btn-sm classes.

Cards

Cards are flexible and extensible content containers and one of the most commonly used components in Bootstrap. You can define headers, footers, a wide variety of content and powerful display options for cards. They are built with Flexbox and have no margin by default. Here is what a card structure looks like:


<div class="card" style="width: 18rem;">
    <img class="card-img-top" src=".../100px180/" alt="Card image cap">
    <div class="card-body">
        <h5 class="card-title">Card title</h5>
        <p class="card-text">Some quick example text to build on the card
            title and make up the bulk of the card's content.</p>
        <a href="#" class="btn btn-primary">Go somewhere</a>
    </div>
</div>

Content types

Cards support a variety of content like images, text, list groups and more. Here are some of the most important Content Types for Cards:

Body

You can use body elements by assigning the .card-body class to a <div> element. You will usually use this when adding padded content (like text, title, links etc.) to the card.

Titles, text and links
These are the most common content types. You can add them to your card like this:
  • By adding .card-title to a <h*> element you can add a title to your card. Subtitles are added with .card-subtitle also used on a <h*> element.
  • With .card-text added to a <p> element you can add text to your card
  • Links are added by using .card-link on an <a> element
Images

With the .card-img-top class you can add an image to the top of the card.

Header, Footer

You can add optional headers and footers to your cards. You can add them by assigning .card-header, .card-footer to either <div> or <h*> to additionally style the header/ footer.

Sizing

Since cards by default don't have a defined width you can define the width with custom CSS and grid.

Sizing with bootstrap grid

By wrapping the cards in rows and columns you can utilize the Boostrap grid system to change their size.

Sizing with utilities

By using the custom sizing utilities offered by bootstrap you can quickly set a card's width. (For example by adding .w75 to the card's <div>.

Card groups

Bootstrap not only offers many options to style content within cards, but also allows you to group cards together into card groups. You can start using them with the .card-group class within a <div> that wraps the card elements. With this your cards will all be a single attached element with equal height and width for each card. For now these layout options are not responsive yet, so use with caution. To use a card group that is not connected to one another simply use the .card-deck class for your card group wrapper.

Navbar

Bootstrap also comes with its own navbar styling and utilities. To start using navbars add the .navbar utility class to your <navbar> element. From there you can use one of the mutliple built in sub-components with these utility classes:

  • .navbar-brand can be applied to most elements but anchors probably work best
  • .navbar-nav for the actual navigation menu, supports dropdowns
  • .navbar-toggler intended to be with Bootstrap's collapse plugin
  • .navbar-text for vertical centering of strings
  • .collapse.navbar-collapse for grouping and hiding navbar content based on a parent breakpoint
  • .navbar-scroll used to set a max-height to allow scrolling of expanded navbar content

Color schemes

To mix things up you can use a few pre-defined styles for your navbar. By using theming classes like navbar-light and navbar-dark and combining them with background-color utilities you can create your own navbar styling.

Placement

To fix the navbar to the top you can use .fixed-top which uses fixed positioning to place the navbar at the top of the viewport, removing it from the normal flow of the document. Another option is .sticky-top which uses position: sticky to fix the navbar to the top.

Utilities

Borders

Bootstrap also comes with pre-defined border utilities:

  • .border, .border-0
  • .border-top, .border-top-0
  • .border-right, .border-right-0
  • .border-bottom, .border-bottom-0
  • .border-left, .border-left-0

The classes with the 0 at the end represent the inverse of the regular border utility classes (F.e. .border-top-0 adds a border to every side except the top). You can add a border radius with the following classes:

  • .rounded
  • .rounded-top
  • .rounded-right
  • .rounded-bottom
  • .rounded-left
  • .rounded-circle
  • .rounded-0

Colors

Bootstrap offers a collection of utility classes that are meant to convey the meaning of the content through their color/ styling. Here is a list of all of them:

  • .text-primary
  • .text-secondary
  • .text-success
  • .text-danger
  • .text-warning
  • .text-info
  • .text-light
  • .text-dark
  • .text-body
  • .text-muted
  • .text-white
  • .text-black-50
  • .text-white-50

You can also use the same styles for background colors. Simply use .bg-primary/secondary etc... to achieve the same styles for your background.

Display

Boostrap comes with utility classes to add any display value to your elements like flex, grid, block, none etc... To add them use the following classes:

  • .d-{value}
  • .d-{breakpoint}-{value}

As you can see you also have the option to set the display value different at any breakpoint. You can use this concept to hide elements at certain viewport sizes for mobile friendly web apps.

Flex

Bootstrap offers utility classes to configure your flex elements. Here is a list of the most important ones:

  • .flex-row, flex-column, flex-{breakpoint}-row-reverse etc... to set the flex direction
  • .justify-content-start, .justify-content-center, .justify-content-{breakpoint}-end etc... to set the the alignment along main axis
  • .align-items-start, .align-items-center, align-items-{breakpoint}-end etc... to align items on the secondary axis
  • And many more.

To get a full list of all available utility classes for flex in bootstrap click here to get to the official documentation.

Spacing

Bootstrap offers many options to adjust the margin/ padding of any element with shorthand classes. These classes are built from a default Sass map which range from .25rem to 3rem. To apply spacing you use a standard format to define property, sides, an optional breakpoint and size:

  • {property}{sides}-{size} for xs
  • {property}{sides}-{breakpoint}-{size} for sm, md, lg and xl

For property you can use m, p which stand for margin and padding (You can also specify the side by adding it to m and p like this: mt, mb, ms, me). If you want to know about all the options in specific here is a link to the official documentation that shows you all the available options for properties, sides and sizes.

Position

To quickly set the positon you can use the Boostrap utility class .position-{position-type} (like relative, absolute, fixed etc...). For sticky, fixed you can add sticky-top/ bottom to quickly position your elements to the top/ bottom of the viewport.

Sizing

For quickly setting the height and width of elements you can use the sizing utilities:

  • .w-25/50/75/100/auto for width
  • .h-25/50/75/100/auto for height
  • .mw-25/50/75/100/auto for max-width
  • .mh-25/50/75/100/auto for max-height
